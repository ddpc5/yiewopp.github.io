13. Exercise: How is the Data Modeled?

In this scenario, you're still the product manager for a company developing smartwatches, but now you're focused on global expansion initiatives.

The company has launched in the U.S. but is looking to expand globally out in Asia. Note that the current physical limitation of these smartwatches is that you need an associated smartphone from the same company in order for the smartwatch to be functional. In order to start your analysis, you’ll need to know the potential market size, which at the maximum, is based on how many smartphones have been distributed and used globally. The first question you’ll need to answer is: How many smartphones were sold in 2019 in Asia?

Hints:

  • You don't need to reference an external mapping online that links country to region. It's already in one of the tables.
  • For these datasets: Any table appended with _f in the table name is a fact table. Any table appended with _d in the table name is a dimensional table.
  • For these datasets: Any field appended with _ts denotes that field as a timestamp.
  • Note that the answer you get is just applicable for your company’s smartphone sales here, which are going to be extremely small compared to all smartphones sold in real life in Asia in 2019.

There will be 3 tables available in the Udacity SQL Workspace for this exercise.

  • sales_f
  • product_d
  • country_d
