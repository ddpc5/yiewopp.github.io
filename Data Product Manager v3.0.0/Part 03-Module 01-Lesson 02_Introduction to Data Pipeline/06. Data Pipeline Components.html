Components Recap

Data Pipeline Components

  • Data Origin: Where data gets into the pipeline
  • Data Processing: Ingestion, transformation, and delivery of data
  • Data Storage: Data Persistence
  • Data Destination: Where data is delivered

Other things to consider while designing a Data Pipeline are:

  • Dataflow design: the dataflow defines how data moves through the pipeline, the sequence of processes, and data stores.
  • Workflow design: the workflow manages the sequencing and dependencies of the processes in the pipeline.
  • Monitoring the pipeline to ensure it is healthy, efficient, reliable and performs as required
