Structured Data

Raw Data

"Raw data, also known as primary data, is data (e.g., numbers, instrument readings, figures, etc.) collected from a source. In the context of examinations, the raw data might be described as a raw score."

Structured Data

Structured data is quantitative data, these are facts and numbers that can be collected, exported, stored, and organized to run analysis on.

  • Quantitative Data
  • Model Designed by DB Engineers
  • Entity Data stored in Rows and Columns
  • Relational
  • SQL
  • OLTP Data

Semi-Structured Data

Semi-Structured Data

Semi-structured data does not conform to relational databases such as Excel or SQL, but has some level of organization like tags.

  • Not strictly Relational
  • With Processing it can be stored in a Relational DB
  • NoSQL DB
  • CSV, XML and JSON
  • Flat Files and API Data

Unstructured Data

Unstructured Data

Unstructured data is qualitative data. Example: Customer reviews. What customers are saying is important but there is no easy way to run analytics on these comments.

  • Qualitative
  • No Model
  • Not Relational
  • Data Lakes or NoSQL
  • Audio, Video or PDFs
  • User Reviews
