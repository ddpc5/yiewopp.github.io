15. Glossary & Resources

Key Terms

Key Terms

Event is an interaction or an action at a particular moment by a user, a system or a sensor.

Event Data is data generated by events is called event data. Event data primarily comprises action, timestamp and state.

KPI or Key Performance Indicator is a performance measurement for an activity or product.

Active Users is the number of users actively using an app or website.

Session Length is the amount of time a user spends on the app or website in one session.

Click Through Rate is the percentage of users who clicked on an ad or banner.

Bounce Rate is the percentage of users leaving a website after viewing only one page is called bounce rate.

Conversion Rate is the percentage of users completing a desired action.

ERP is usually referred to as a category of business management software — typically a suite of integrated applications — that an organization can use to collect, store, manage and interpret data from many business activities.

CRM: Customer relationship management (CRM) is an approach to manage a company's interaction with current and potential customers. It uses data analysis about customers' history with a company to improve business relationships with customers, specifically focusing on customer retention and ultimately driving sales growth.

Database: A database is a collection of related data that represents some aspect of the real world. A database system is designed to be built and populated with data for a certain task.

DBMS: Database Management System (also known as DBMS) is a software for storing and retrieving users' data by considering appropriate security measures. It allows users to create their own databases as per their requirement.

Data Warehouse: Data warehouses are a type of DB, which is often used to store very large amounts of data.

API: An application program interface (API) is a set of routines, protocols, and tools for building software applications. Basically, an API specifies how software components should interact.

Flat File: A flat file contains records that have no structured interrelationships. A flat file typically consists of a text file, from which all word processing or other structure characters or markup has been removed.
