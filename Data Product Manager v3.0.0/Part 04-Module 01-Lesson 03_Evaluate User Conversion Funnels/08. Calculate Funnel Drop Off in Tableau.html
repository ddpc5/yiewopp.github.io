08. Calculate Funnel Drop Off in Tableau

Calculate Funnel Drop Off in Tableau

ND030 C3 L2 07 Calculating Funnel Drop Off In Tableau Video

Summary

To calculate a funnel drop off rate using Tableau, we can follow 4 steps:

  1. Review available data to look for the pieces of data that correspond to a user’s steps in the funnel. For example, data showing a user visited a page at 1 pm might be a step in the awareness stage; data showing the user clicked on a button at 1:01 pm might be a step in the evaluation stage.

  2. Determine users' steps and put them in the correct order, from the first action to the final goal. Note that not all steps a user takes need to be included in a funnel. For example, some steps are optional and should not be included.

  3. Count the number of unique sessions where users take each step. A session is a period of time in which a user interacts with a product. If a user interacts with the product at different times, or on different days, each of these would be a unique session. The number of unique sessions might be calculated in the following ways

  • If every user could only interact with the feature one time (such as a feature that is only shown once to users, on the first login), count the user ID would be the same with count the unique sessions.
  • If a user could have had multiple sessions where they interacted with a feature (such as a feature that appears every time a user logs in or a feature can be used multiple times a day), you need to use count the distinct session not the user ID
  1. Calculate the percent change (i.e. drop off) between steps. This means evaluating the relative size of the number of users who moved from one step to another. In a funnel, as users drop out and the number of users that make it to each subsequent step gets smaller, the calculation will show a percent decrease between each step. By looking at those steps with a high percent change, you can identify areas that may need investigation and further improvement.

Step 1: Review the available data

The data is for a website that sells items. There are two dimensions: user ID and Pages. The name of the pages corresponds to a specific action taken by users.

Step 2. Put the steps in order from the first step to the final goal step

After reviewing each dimension of the data, you can see that the name of the pages (home page, payment confirmation page, payment page, and search page) are the steps a user would take in a purchase funnel.

The correct order should be home page > search page > payment page > payment confirmation page

Step 3: Count the unique sessions for each step

By comparing the number of records for each page with the number of user ID, you know each user only went through the funnel once since the numbers are identical. So number of records can be used for calculating the dropoff.

  • If every user could only interact with the feature one time, you can use anyone from COUNT user, COUNT (Distinct) user, COUNT session, COUNT (Distinct) session and get the same result, as you will never double-count a user.
  • If a user could have had multiple sessions where they interacted with a feature, you should use COUNT (Distinct) session.

**Step 4: Determine the percent change or dropoff rate between each step. **

To do this, we apply a table calculation for percent change. In Tableau, you can apply Quick table calculation > percent difference to the numbers generated in step 3. In this example, the highest level of drop off occurs in the final step, between payment and payment confirmation, at 92%, which indicates the product manager probably needs to find ways to make payment easier.
