16. Solution: Instrument and Collect Data at Feature-Level

Instrument and Collect Data at Feature-Level Solution

Instrumenting a New Referral Feature

ND030 C3 L1 16 Collecting Additional Data For KPI Calculations Exercise Solution Video

Summary

Question 1: Choosing Data to Track

Whenever launching a new feature (or test), it is important that the success of the feature can be verified. In the case of a referral feature, you should ensure you can track the entire referral process:

  • which users send invites
  • how many users send invites
  • how many invites an individual user sends
  • if an invite results in a new customer
  • which users arrive via referral (this will allow you to calculate the the LTV of the new customers that arrive via referral vs. the LTV of traditionally acquired customers)

Note: if different methods of referral are in the initial tests, make sure different methods of referral can be tracked separately. For example, you might give users a way to share on social media that generates a different code than if a referral is sent via email. This will allow you to compare the above data by method to see if one form of sharing is typically more impactful.

Question 2: Future Data Collection Needs

When designing new features, it is important to instrument with the near-term evolution of the feature in mind. If you know incentives might be offered to the referrer in the future, you should ensure that each code and referral is associated with a particular user. More than that, even if the new feature is never launched, important KPIs such as how many users send invites or which users sent invites could be lost if the codes are universal.

When instrumenting features, knowing exactly who has done what is crucial to measuring success. This also makes reaching out to users and non-users of the feature for feedback simpler, as you will know which users have used the feature.
