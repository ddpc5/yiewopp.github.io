13. Instrument and Collect Data at Feature-Level

Instrument and Collect Data at Feature-Level

Feature-Level Data Collection

ND030 C3 L1 13 Collecting Additional Data For KPI Calculations Video Part 1

Summary

Product managers also help determine, create, and launch new features within a product. KPIs are just as important at the feature level as the product level. Before launching a new feature, you need to think about what data to collect at the feature-level and how to instrument the feature to collect the data.

Data collection

While the Pirate Metrics framework can provide a holistic view of a product’s performance, it can also be leveraged to examine the success of a new feature. At the feature-level, data should be collected including:

  • Acquisition: who saw the feature
  • Activation: who used the feature
  • Retention: how often users returned and re-used the feature
  • Revenue: how much can be made from the feature

Besides data from the Pirate Metric framework, it is also important to collect customer data. This data includes:

  • Which customer uses the feature and which don't. By tracking this, you can identify if the feature is used differently among different customers and you can know who to talk to to get user feedback after launch.
  • When a user uses the feature. This data helps you understand why a customer uses the feature and helps you build new features that meet users' needs.
  • How a user uses the feature (event-level data and session-based data) such as the user's operating system or other information that can change from session to session. This data tells you if the feature is used differently. It provides valuable insights into further improvement.

Feature-Level Metrics and Instrumentation

ND030 C3 L1 14 Collecting Additional Data For KPI Calculations Video Part 2

Feature-level metrics

The feature-level data can then be used to evaluate if the feature is successful or not. There are two common metrics that you can calculate from the feature-level data.

The percent of feature users out of total product users

This percentage measures how many users actually use a feature after seeing it. It tells you how widespread the feature usage is among all of a product’s active users.

Feature usage frequency and duration

This data measures how frequently a user engages with the feature and on average, how much time customers spend on the feature during each session. It provides information about how the feature is being used.

Note that whether the frequency and duration of feature usage are “good” or “bad” depends entirely on the context of the product and the feature.

When the frequency and duration don’t match product expectations or the frequency & duration of similar features already launched in the product, this is a sign that further investigation and research is needed.

Feature instrumentation

After knowing the data you want to collect at the feature-level, you need to instrument the feature to ensure it can be collected. There are a few best practices in feature instrumentation.

  1. Keep in mind the data you want to collect when making decisions on how to build a feature. Common data that should be collected: who, when, how the feature is used, and different events/actions.
  2. Test instrumentation before launch. You can simulate the data that will be collected and check if the data can answer every question you have in mind.
  3. Define and document any new events/data created. Describe exactly how a piece of data is triggered. Data isn’t helpful if you don’t know the meaning of it.
  4. Bottom line: Keep in mind your future self. Think about the data you want to collect in three months from now and start to plan and instrument it now.

New Feature Example

Summary

While features don't typically need additional revenue instrumentation and some features may not need retention instrumentation, the retention and revenue of a product can be highly relevant with feature-level success. By adding instrumentation and collecting data at a feature-level, product managers can compare feature usage to product KPIs. For example: is user satisfaction different among users who use and not use the feature? Are users who use the feature retains longer than those who don't use the feature? Answering these questions can help product managers find areas for further improvement.
