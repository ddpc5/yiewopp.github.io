Data Warehouse vs Data Lake

Data Warehouse vs. Data Lake

When should you use Data Warehouses, and when should you use Data Lakes?

Type of Data

When you have unstructured and structured data from various company data, sources use a data lake. When you have Historical data that has been structured to fit a relational database schema, use Data Warehouses.

Purpose

When you need cost-effective big data storage, use a Data Lake. When you need to store data to run analytics for business decisions, use Data Warehouses.

Users

Data Lake users are often data scientists and engineers, whereas Data Wharehouse users are data analysts and business analysts.

Workload

Data Lakes are suitable for storing massive data and big data analytics, like deep learning and real-time analytics. Data Warehouses are ideal for a smaller set of data compared to DL and are used for typical read-only queries that aggregate or summarize data.

Size

Data Lakes store all data that might be used and can take up to petabytes! Data Warehouses store only data relevant to the analysis.

Neither/Both?

Sometimes neither or both are better, while there are specific scenarios for each one, most enterprises adopt a hybrid approach. A business may catch all their data in a data lake and build ETL on top of it to feed DWH. For a small business, you might not need either.

DW vs DL Quiz

Which of the following is True for Data Warehouses(DWHs) and Data Lakes(DLs)?
SOLUTION:
  • For DLs the storage is cheaper so that whatever data we want can be stored. Comparatively DWHs are expensive where only required and relevant data is stored.
  • DWHs are best when data requirements are clearly defined and data is not unstructured or semi-structured. DLs can be used in other cases.
