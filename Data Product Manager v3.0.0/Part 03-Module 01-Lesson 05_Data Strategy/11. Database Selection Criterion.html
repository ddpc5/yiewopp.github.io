Database Selection Criterion Recap

Choosing the right Database

Here are some tips.

  • Look at the possible structure of your data?
    • clearly relational such as transactional data
    • can it be mapped to key-value pairs like user comments or ratings?
  • What is the size of the data?
    • Big data or regular data.
  • What are the latency requirements?
    • used for reporting where data can be made available in minutes or hours
    • Is it needed for real-time use cases such as upsell/cross-sell?
  • What are the core requirements of this DB?
    • are the operations read-heavy or write-heavy
    • will it be needed for search operations

SQL Databases or Relational DB

  • Most mature and stable category of Databases -> Security
  • Structured Data
  • Financial applications or defense

Document Oriented

  • Schema Free
  • Semi-structured or Unstructured Data
  • Content Management System: WordPress

Columnar DB

  • Eventual Consistency
  • Query on subset of Column
  • Data Science

Key Value

  • Simple key-value data
  • Unstructured Data
  • Customer Reviews

Graph

  • Graph like data
  • Connections, nearest neighbor -> Focus on relationships
  • Social Networks

Selection Quiz

Which of the following is not a database selection criterion
SOLUTION:
  • When is data generated - Monday vs Friday.

Database Selection Quiz

Which of the following are something to consider for a database selection criterion
SOLUTION:
  • Structure of data generated
  • Timeliness requirements
  • Size of the data
  • Use case of the database
