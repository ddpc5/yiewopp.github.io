Database Definition

A database is an organized collection of data, generally stored and accessed electronically from a computer system.

A data warehouse is a central repository of information that can be analyzed to make better-informed decisions. Data flows into a data warehouse from transactional systems, relational Databases, and other sources, typically on a regular cadence.

To summarize, Data Warehouses are required to store large quantities of historical data and provide fast, complex queries across all the data, mostly using OLAPS. Databases focus on storing current transactional data to enable fast access to specific data for business processes. This is how OLTPs work.

Types of Database

Databases can be categorized into two major classes SQL Databases and No SQL Databases.

SQL Databases are the classic category of Databases they store data in tables, data is relational in nature and can be queried using SQL. Schemas need to be defined. A Schema is nothing but an organization of data as a blueprint of how the Databases is constructed.

Common SQL Databases are MS SQL Server, Oracle and MySQL

No SQL Databases

  • Document oriented

Document oriented Databases do not have a fixed schema. Data is stored in JSON documents, each document can have different fields common examples are Mongo DB, CouchDB or Documentdb

  • Columnar Databases

In columnar Databases data is stored in columns instead of how it is stored in rows in relational DBs. Column based queries can run fast on these DBs. It is much easier in columnar Databases to add columns than their counterpart relational DBs. One common example of the most popular columnar Databases is Cassandra.

  • Key-Value Databases

In key-value Databases, values are stored with unique keys. These Databases are much simpler and capable of reads and write. Examples of key-value Databases are Redis and Amazon DynamoDB.

  • Graph Databases

In graph Databases, data is stored as a network. These Databases focus on connections between data points and are used for the analysis of relationships between data points. Examples of graph Databases are Infinite Graph and Neo4j.
