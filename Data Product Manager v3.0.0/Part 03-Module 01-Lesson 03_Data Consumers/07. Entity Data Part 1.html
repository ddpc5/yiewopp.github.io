What is Entity Data?

NDO30 DPMND C2 L2 06 Entity Data Intro

Entity Text

Entity Data

An Entity is a unique object such as a customer, product, order, etc.

Entity Attributes are the properties that describe it.

For example, we can have three entities in any business:

  • the customer
  • the product
  • the order

The attributes for a Customer could be:

  • Customer id
  • name
  • address
  • email
  • phone

Entity data is stored in a table, for every entity there is a separate table, and every table has columns that describe these entities. One row is dedicated to recording one type of each entity.

The concepts discussed here are fundamental to the understanding of relational databases. Relational databases have data stored in tables that are related to each other through keys.

This type of entity data helps capture our application's current state, such as our customers' information or what the product range is.

The drawback of the entity data model is that the joins might get complicated and introduce latency.

The advantages are the removal of data duplication and a simple, easy to understand basic structure.
