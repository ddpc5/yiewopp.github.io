Relating and Normalizing Entity Data

Entity data captures the current state of an application, such as - who our customers are or what products we have.

To summarize, Customer ID, Product ID, and Order ID are our typical primary keys in their respective tables.

And Customer ID and Product ID would be foreign keys in an order table.

Keys help us in establishing relationships between tables. Once we have relationships established, we can run complex queries to join data from different tables.

The concepts discussed here are fundamental to the understanding of relational databases.

  • Relational databases have data in tables
  • Tables are linked through keys
  • Data in these tables is normalized.

In our example, if we want to create a buying pattern for Jon, we can join the order table and product table to see Jon prefers a red color product and is a frequent buyer. This information is essential for marketing to advertise to Jon the products that fall in the red color palette. This information is also vital for inventory management teams as they can predict the demand for red-colored products and have inventory accordingly.

We have a minimal data set here. Generally, these insights are derived from tables that could have millions of rows.

Normalization refers to eliminating redundancy. Each table should have data around a specific topic. This eliminates modification complexities and saves on storage.
