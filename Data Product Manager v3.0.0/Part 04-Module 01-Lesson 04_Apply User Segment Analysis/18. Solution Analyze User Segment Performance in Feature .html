Solution: Random Song Feature Analysis

ND030 C3 L3 14 Analyzing Product & Feature Usage In Tableau Exercise Solution Video

Error in the video: the instructor said: "Since these are first-time users, there will be only one session ID for every User ID and vice versa". It is not correct. Since these first-time users can use the features multiple times, each User ID may have several session IDs.

For the event time segment: you will follow the same procedure. See the screenshots below for the answers and explanations.

Summary

1) Determine the number of users that used each feature

Looking at the data file, we can see three "event types" (landing_page, play_random_song, and play_requested_song). We know that the new feature of the music app was the ability for new users to play a random song, so all play_random_song event types likely represent a user making use of this new feature. In contrast, the "play_requested_song" represents the new user using the original feature set.

To see the number of users using each feature, one has to count the number of users associated with each event type. Since the users could use the features multiple times in one visit, so we will want to use COUNT user ID (distinct) to ensure we don't count the same user multiple times.

By counting the number of unique user IDs associated with the two event types in Tableau, we can see 1,709 users played a random song and only 1,045 users played a specific song during their visit.

2. Based on available segments, analyze the data to determine if the use of the two features was different among segments. Explain your answer using numbers as evidence of these findings.

Since we are interested in the actual use of the two features not the number of unique users. You should use COUNT when calculating the relative percentage of use.

The data provides three potential elements of demographic or behavioral distinction: gender, device, and date. By looking at the count of the "requested" and "random" events broken down into groups for each segment, we can calculate the percent of use by each user group. This allows us to determine if any type of user used one of the two features more often.

As seen in the tables below, both genders have about the same ratio (of all the times either feature was used, 67-69% times were the random feature and 30-32% were the requested feature). Similar ratios are seen among all devices types as well (of all the times either feature was used, about 67-68% were the random feature and 30-32% were the playing a requested song). When looking at the differences among feature usage by day, a similar pattern is seen. This implies that none of the behavioral and user characteristics tracked contributed to one of the features being more popular with a particular group.
