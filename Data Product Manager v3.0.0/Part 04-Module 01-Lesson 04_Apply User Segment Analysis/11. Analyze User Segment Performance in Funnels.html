11. Analyze User Segment Performance in Funnels

Analyze User Segment Performance in Funnels

ND030 C3 L3 09 Applying Cohort Analysis To Funnel Analyses In Tableau Video

Summary

Segment analysis of funnels can reveal users who drop off or convert. Product managers who perform such analysis can learn why a difference in performance occurs for different user types and will likely learn valuable information about how to optimize a product.

Segment analysis of funnels follows four basic steps:

  1. Create the overall funnel and count the number of users who complete each step.

  2. Determine which user segments are available in the data. These user segments are the ones you will apply to the funnel analysis.

  3. Calculate the funnel dropoff for each group within a segment.

  4. Look for groups within a segment with a drop off rate that is noticeably higher or lower than average.

If a particular group of users isn’t getting the experience they need, this will likely be reflected in a drop out at a rate that is higher than average. Once such a pattern is identified, product managers can talk to those users and run experiments to find ways to optimize their user experience.

Summary

Step 1: create the overall funnel and count the number of users per session who completed each step

For this funnel, a user begins by landing on the home page and completes the funnel when a purchase is made. Right now the funnel shows the number of users who completed each step.

Note: in this example, the data set only showed the first time any given user went through the funnel. Each user will only ever have taken a step one time. Therefore, we can use the count of users OR the distinct count of users OR the number of unique sessions and get the same result.

Step 2: determine which user segments are available in the data

For this example, let’s look at the segment of devices. By adding the devices to the funnel analysis, we can see the device segment is divided into groups of users on mobile devices and desktop.

Step 3: calculate drop off for each group in a segment.
Use a table calculation of percent difference to calculate the dropoff and select table down.

Step 4: look for groups within a segment with a drop off rate that is noticeably higher or lower than average.

Looking at the data, we can see mobile users have a 10% lower drop off rate between the search and payment pages than that of the desktop users. They also have a 5% lower drop off rate between the payment and payment confirmation pages than the desktop users.

The findings may indicate areas for desktop experience optimization. The product manager should talk to users and/or review the desktop design to determine what may be causing the higher drop off rates.
