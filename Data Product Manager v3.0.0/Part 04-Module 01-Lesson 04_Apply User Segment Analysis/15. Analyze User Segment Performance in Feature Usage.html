Analyze User Segment Performance in Feature Usage

ND030 C3 L3 12 Analyzing Product & Feature Usage In Tableau Video

Summary

Segmentation of feature usage data shows whether a feature is being adopted at the same rate or used in the same manner by different types of users. Differences in feature use among segment groups can lead to realization that a feature needs to be made more appealing to the under-using segment group.

It is also worth noting that feature usage itself can be a behavioral segment. KPIs from customers that use a feature can be compared to those that don’t, or customers that use a feature more frequently can be compared to those who use it less frequently.

Summary

First, we examined the number of feature users compared to the number of product users. We can see 40 of the 70 product customers have used the attendance feature. That is a larger percentage of customers. But we still need to see if the different user groups use the feature at the same frequency.

In this data set, we have the job dimension (elementary school teacher vs. middle school teachers). We can see that 2/3 of elementary school teachers and 1/2 of middle school teachers are using the attendance feature. Now that we’ve identified this difference in the data, we can talk with customers and run experiments to determine why feature usage is different and ways to improve it.

Comparing the Use of Multiple Features

In the example above, we analyzed the use of one feature, an attendance tracker, in reference to the total number of users of a product. Sometimes, product managers want to compare the use of two or more features. For example, in the exercise, you want to see if users use "play a random song" feature differently from the "play a requested song" feature.

To compare two features, you can follow the same process as outlined in the walkthrough and look at the number of feature users compared to the total number of product users. However, to better compare two feature usage, you can calculate the relative percent of users who used a feature.

Analyze features used only ONCE

  1. Place the events that correspond to a feature being used in Tableau
  2. Place of COUNT or COUNT (distinct) of users to see the number of unique users who used the feature. The two methods result in the same number since the feature can only be used once.
  3. Use a Quick Table Calculation > Percent of Total to see the relative number of uses of one feature compared to the other.

For example, if a product has 200 users, Feature A was used by 100 users and Feature B was used by 50 users, Feature A would represent 66% (100 out of 150) and Feature B would represent 33% (50 out of 150) of the total number of users who used either features. Note: the percentage is calculated against the total number of users who used either feature (100+50) not the total number of users who use the product (200). You can see that feature A is twice as popular as feature B.

Analyze features used MULTIPLE times

  1. Place the events that correspond to a feature being used in Tableau
  2. Place COUNT (distinct) of users to see the number of unique users who used the feature at least once.
  3. Use a Quick Table Calculation > Percent of Total to see the relative number of uses of the features. Note: since COUNT and COUNT (distinct) now results in different numbers, when to use COUNT and COUNT (distinct) depends on the question. See the examples below.

For example, if feature A was used by 50 users, feature B was used by 25 users, and feature A was used 100 times, feature B was used 300 times.

If using COUNT, we count all uses of the feature, including multiple uses by the same person. We will see feature A was used 100 times and feature B was used 300 times. Feature A would have gotten 25% (100/400) and Feature B would have gotten 75% (300/400) of the total uses of either feature. Feature B is used three times as often as feature A.

However, same scenario but using COUNT (distinct), it counts the unique users of each feature. We will see feature A was used by 50 users at least once and feature B was used by 25 users at least once. Feature A would represent 66% (50/75) and feature B would represent 33% (25/75) of the total users who used either feature at least once. You can see that feature A has twice more users compared to feature B.

To summarize, if you are interested in the actual use of different features, COUNT is a better choice. However, if you are interested in the number of users who uses different features, you should use COUNT (distant).
